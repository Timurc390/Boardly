# Security Policy

## Supported Versions
Security updates are provided for the latest state of the `master` branch.

## Reporting a Vulnerability
If you discover a security issue, please do not open a public issue first.

Please report it using one of the following:
- GitHub private vulnerability reporting (preferred).
- Direct contact with repository maintainers.

Include:
- a clear description of the issue
- impact and affected area
- steps to reproduce (if possible)
- suggested fix (optional)

We will acknowledge valid reports and work on a fix as quickly as possible.
